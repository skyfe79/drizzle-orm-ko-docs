# 기존 프로젝트에 Drizzle와 SQLite 시작하기


<Prerequisites>  
  - **dotenv** - 환경 변수를 관리하기 위한 패키지 - [자세히 보기](https://www.npmjs.com/package/dotenv)
  - **tsx** - TypeScript 파일을 실행하기 위한 패키지 - [자세히 보기](https://tsx.is/)
  - **bun** - 자바스크립트 올인원 툴킷 - [자세히 보기](https://bun.sh/)
  - **bun:sqlite** - 고성능 SQLite3 드라이버의 네이티브 구현 - [자세히 보기](https://bun.sh/docs/api/sqlite)
</Prerequisites>

<FileStructure />


#### 1단계 - 필요한 패키지 설치하기

<Npm>
  drizzle-orm dotenv
  -D drizzle-kit tsx @types/bun
</Npm>

- **drizzle-orm**: 데이터베이스 ORM 라이브러리
- **dotenv**: 환경 변수 관리를 위한 패키지
- **drizzle-kit**: Drizzle ORM 개발 도구
- **tsx**: TypeScript 실행 환경
- **@types/bun**: Bun 타입 정의 파일

위 명령어를 실행하면 프로젝트에 필요한 패키지들이 설치됩니다.


#### 2단계 - 연결 변수 설정하기

<SetupEnv env_variable='DB_FILE_NAME' />

<Callout type='info' title='중요'>
예를 들어, 프로젝트 루트에 SQLite 데이터베이스 파일이 있다면 아래 예제를 참고할 수 있습니다:
```plaintext copy
DB_FILE_NAME=mydb.sqlite
```
</Callout>
